# Knowledge file structure

Use this structure for every new knowledge file. Sections can be renamed to fit the topic (e.g. "Key principles", "Case studies"); keep "Description" at the top and a clear hierarchy of headings.

```markdown
# [Knowledge file title]

## Description
[Short description of what this file contains]

## Main sections
### [Section 1]
[Content]

### [Section 2]
[Content]

## Practical examples
### Example 1
[Example and how to apply it]

## Links and resources
- [Link or resource] — [description]

## Notes
- [Key takeaway or reminder]
```
