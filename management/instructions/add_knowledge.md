# Инструкция: Добавление знаний к агенту

## Общие принципы

Эта инструкция предназначена для LLM-агентов по добавлению новых знаний к существующим агентам в системе.

## Структура знаний

Знания агента хранятся в папке `agents/[agent_name]/knowledge/` в виде отдельных Markdown файлов.

## Пошаговая инструкция

### Шаг 1: Создание файла знаний
- Создайте новый файл в `agents/[agent_name]/knowledge/`
- Используйте описательные названия файлов
- Следуйте формату из шаблона `management/templates/knowledge.md`
- Заполните файл предоставленной пользователем информацией

**Примеры названий файлов:**
- `marketing_strategies.md` - маркетинговые стратегии
- `best_practices.md` - лучшие практики
- `tools_and_resources.md` - инструменты и ресурсы
- `case_studies.md` - кейсы и примеры
- `templates.md` - шаблоны и формы

### Шаг 2: Заполнение файла знаний
Используйте структуру из шаблона `management/templates/knowledge.md`:

```markdown
# [Название файла знаний]

## Описание
[Краткое описание содержимого]

## Основные разделы
### [Раздел 1]
[Содержимое раздела]

## Практические примеры
### Пример 1
[Описание примера]

## Полезные ссылки и ресурсы
- [Ссылка 1] - [описание]

## Важные заметки
- [Заметка 1]
```

### Шаг 3: Обновление индекса в role.md
**ОБЯЗАТЕЛЬНО**: Добавьте новый файл в раздел "База знаний" файла `role.md`

**Формат добавления:**
```markdown
## База знаний

### [название_файла.md]
Краткое описание содержимого файла и когда его использовать.

### [следующий_файл.md]
Краткое описание содержимого файла и когда его использовать.
```

**Пример:**
```markdown
## База знаний

### marketing_strategies.md
Основные маркетинговые стратегии и подходы. Используй при планировании кампаний и разработке маркетинговых планов.

### best_practices.md
Лучшие практики в маркетинге. Применяй при создании контента и оптимизации процессов.
```

## Правила именования файлов знаний

- Используйте описательные названия
- Разделяйте слова подчеркиваниями
- Используйте английские названия
- Избегайте специальных символов

**Хорошие примеры:**
- `content_strategies.md`
- `social_media_tips.md`
- `analytics_metrics.md`

**Плохие примеры:**
- `file1.md`
- `новый файл.md`
- `tips & tricks.md`

## Организация знаний

### Логическая группировка
Группируйте связанные знания в отдельные файлы:
- Стратегии и подходы
- Инструменты и ресурсы
- Примеры и кейсы
- Шаблоны и формы

### Размер файлов
- Держите файлы знаний в разумных пределах (до 2000 слов)
- При необходимости разбивайте большие темы на несколько файлов
- Каждый файл должен быть самодостаточным

## Важные напоминания

1. **Всегда обновляйте индекс** в `role.md` при добавлении новых знаний
2. **Следуйте шаблону** из `management/templates/knowledge.md`
3. **Проверяйте актуальность** информации в файлах знаний
4. **Используйте четкую структуру** заголовков и разделов
5. **Добавляйте практические примеры** для лучшего понимания

## Шаблон

Используйте файл `management/templates/knowledge.md` как основу для создания новых файлов знаний.
