# Инструкция по обработке книги и созданию базы знаний

## Описание процесса

Эта инструкция описывает двухэтапный процесс создания структурированной базы знаний из любой книги. Процесс универсален и подходит для книг по любым темам.

## Определение контекста

### Перед началом работы
1. **Уточните у пользователя:**
   - К какому агенту относится книга?
   - Какая тематика книги?
   - Какой тип знаний нужно извлечь?

2. **Если контекст не указан:**
   - Спросите: "К какому агенту и какой теме относится эта книга?"
   - Определите целевую директорию для сохранения файлов
   - Адаптируйте критерии отбора под тематику

### Примеры вопросов для уточнения:
- "Это книга по маркетингу для агента-маркетолога?"
- "Книга по программированию для агента-разработчика?"
- "Книга по дизайну для агента-дизайнера?"
- "Книга по управлению для агента-менеджера?"

## Этап 1: Извлечение принципов из книги

### Цель этапа
Создать единый файл с извлеченными принципами из книги, следуя структуре "Суть → Применение → Практический совет".

### Процесс работы

1. **Получение блоков текста**
   - Пользователь присылает блоки текста из книги по частям
   - Каждый блок может содержать одну или несколько глав
   - Анализируйте каждый блок на релевантность для тематики книги

2. **Извлечение принципов**
   - Ищите практические принципы, применимые к тематике книги
   - Исключайте "воду" и нерелевантную информацию
   - Фокусируйтесь на практическом применении, а не на теории
   - Избегайте повторений - проверяйте, что уже есть в файле
   - Адаптируйте критерии отбора под конкретную тематику

3. **Формат принципов**
   ```
   ### **Название принципа**
   - **Суть**: Краткое описание сути принципа
   - **Применение**:
     - Конкретные способы применения
     - Практические советы
     - Примеры использования
   - **Практический совет**: Один ключевой совет для применения
   ```

4. **Критерии отбора принципов**
   - Принцип должен быть практически применим
   - Должен содержать конкретные данные/проценты (если есть)
   - Должен объяснять "почему" и "как"
   - Должен быть релевантен для тематики книги
   - Адаптируйте критерии под конкретную область знаний
   - **Важно**: Информация должна быть оптимальной для работы LLM-агента

5. **Структура файла**
   - Название: `[название_книги]_extracted.md`
   - Заголовок с описанием источника
   - Раздел "Основные принципы"
   - Заключение с указанием количества принципов

### Примеры хороших принципов

**Хорошо (универсальные примеры):**
- "Позитивное эмоциональное окружение повышает привлекательность товара на 16%"
- "Оптимальная плотность товаров: 1-1,5 модели на квадратный метр"
- "Код должен быть читаемым и понятным для других разработчиков"
- "Пользователи принимают решения в первые 3 секунды взаимодействия"

**Плохо:**
- "Цвета влияют на эмоции" (слишком общо)
- "Важно изучать аудиторию" (банально)
- "Нужно тестировать решения" (очевидно)

## Оптимизация для LLM-агентов

### Цель: Баланс между детальностью и компактностью

**Принципы создания контента для LLM:**

1. **Достаточная детальность**
   - Включайте конкретные данные и проценты
   - Объясняйте причинно-следственные связи
   - Добавляйте практические примеры применения
   - Сохраняйте важные нюансы и контекст

2. **Оптимальная компактность**
   - Избегайте избыточных деталей и "воды"
   - Убирайте повторения и дублирования
   - Фокусируйтесь на ключевых инсайтах
   - Структурируйте информацию логично

3. **Формат для LLM**
   - Используйте четкую структуру (Суть → Применение → Совет)
   - Добавляйте ключевые слова для поиска
   - Создавайте перекрестные ссылки
   - Группируйте связанные принципы

### Примеры оптимального баланса

**Хорошо (детально, но компактно):**
```
### **Периферийное восприятие работает за 100 мс**
- **Суть**: Периферийные визуальные раздражители активируют мозг уже в первые 100 мс через прямую связь с лимбической системой.
- **Применение**:
  - Создавать "эмоциональные аллеи" в интерфейсах
  - Размещать позитивные элементы по краям экрана
  - Использовать периферийное зрение для подсознательного воздействия
- **Практический совет**: Периферийное восприятие работает быстрее сознания - используйте это.
```

**Плохо (слишком кратко):**
```
### **Периферийное восприятие**
- Работает быстро
- Используйте в дизайне
```

**Плохо (слишком подробно):**
```
### **Периферийное восприятие работает за 100 мс**
[Длинное описание нейрофизиологических процессов, история исследований, детальные объяснения анатомии глаза, полный список всех экспериментов...]
```

## Этап 2: Структурирование базы знаний

### Цель этапа
Разбить единый файл на тематические файлы, убрать дублирования и создать перекрестные ссылки.

### Процесс работы

1. **Получение команды**
   - Пользователь дает команду на структурирование
   - Анализируйте весь накопленный файл с принципами

2. **Анализ и группировка**
   - Найдите дублирующиеся принципы
   - Сгруппируйте принципы по темам
   - Определите ключевые тематические области

3. **Создание тематических файлов**
   - Создайте отдельные файлы для каждой темы
   - Используйте единый формат для всех файлов
   - Добавьте перекрестные ссылки между файлами

### Тематические области (примеры для разных областей)

**Для маркетинга/нейромаркетинга:**
1. Эмоциональные основы маркетинга
2. Визуальное восприятие и дизайн
3. Периферийное восприятие
4. Принципы презентации товаров
5. Психология покупательского поведения

**Для программирования/разработки:**
1. Принципы чистого кода
2. Архитектурные паттерны
3. Тестирование и качество
4. Производительность и оптимизация
5. Работа в команде

**Для дизайна:**
1. Принципы визуального дизайна
2. UX/UI паттерны
3. Цветовая теория
4. Типографика
5. Адаптивный дизайн

**Для управления:**
1. Лидерство и мотивация
2. Планирование и стратегия
3. Командная работа
4. Коммуникация
5. Управление проектами

**Универсальный подход:**
- Анализируйте содержание книги
- Определите естественные тематические группы
- Создавайте 5-10 тематических файлов
- Адаптируйте названия под конкретную область

### Формат тематических файлов

```markdown
# Название темы

## Принцип 1

### Суть
Краткое описание сути принципа

### Применение
- Конкретные способы применения
- Практические советы
- Примеры использования

### Практический совет
Один ключевой совет для применения

## Принцип 2
...

## Связанные темы
- [Связанная тема 1](filename1.md)
- [Связанная тема 2](filename2.md)
```

## Критерии качества

### Для этапа 1 (извлечение)
- ✅ Принцип практически применим
- ✅ Содержит конкретные данные
- ✅ Объясняет причинно-следственные связи
- ✅ Релевантен для тематики книги
- ✅ Не дублирует уже имеющиеся принципы
- ✅ Оптимален для работы LLM-агента (детально, но компактно)

### Для этапа 2 (структурирование)
- ✅ Логическая группировка по темам
- ✅ Отсутствие дублирований
- ✅ Перекрестные ссылки между файлами
- ✅ Единый формат всех файлов
- ✅ Практическая направленность

## Примеры команд

### Этап 1
```
"Проанализируй этот блок текста из книги и добавь новые принципы в файл"
"Извлеки ключевые идеи из этой главы"
"Добавь принципы из следующего блока"
```

### Этап 2
```
"Проанализируй накопленный файл и разбей на тематические файлы"
"Структурируй базу знаний по темам"
"Создай финальную структуру базы знаний"
```

## Важные принципы работы

1. **Фокус на практическом применении** - каждый принцип должен быть применим в реальной работе
2. **Избегание дублирований** - проверяйте, что принцип еще не добавлен
3. **Конкретность** - предпочитайте конкретные данные общим утверждениям
4. **Структурированность** - следуйте единому формату во всех файлах
5. **Перекрестные ссылки** - связывайте связанные темы между собой
6. **Оптимизация для LLM** - создавайте контент, который эффективно работает в контекстном окне LLM-агента

## Результат работы

После завершения обоих этапов у пользователя будет:
- Структурированная база знаний по темам
- Практически применимые принципы
- Перекрестные ссылки между темами
- Готовая к использованию база для соответствующего агента
