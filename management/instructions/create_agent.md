# Инструкция: Создание нового агента

## Общие принципы

Эта инструкция предназначена для LLM-агентов по созданию новых агентов в системе управления ИИ-агентами.

**ВАЖНО**: При создании агента обязательно задавайте пользователю подробные наводящие вопросы, чтобы собрать всю необходимую информацию для полноценного агента с базой знаний.

## Структура агента

Каждый агент должен иметь следующую структуру:
```
agents/[agent_name]/
├── role.md             # Роль + инструкции + индекс знаний
└── knowledge/          # База знаний агента
    └── [knowledge_files].md
```

## Интерактивный процесс создания агента

### Этап 1: Сбор информации о роли агента

**Задайте пользователю следующие вопросы:**

1. **Основная специализация:**
   - Как называется роль агента?
   - Какая основная задача агента?

2. **Детализация роли:**
   - Какие конкретные задачи должен выполнять агент?
   - В каких ситуациях пользователь будет обращаться к этому агенту?

3. **Ограничения и принципы:**
   - Есть ли ограничения в работе агента?
   - Какой стиль общения предпочтителен?

4. **База знаний:**
   - Какие знания должен иметь агент для эффективной работы?

### Этап 2: Создание структуры агента

После сбора всей информации:

1. **Создание папки агента**
   - Создайте папку `agents/[agent_name]/`
   - Используйте английские названия в нижнем регистре
   - Название должно отражать специализацию агента

2. **Создание файла роли**
   - Скопируйте шаблон из `management/templates/role.md`
   - Заполните все разделы на основе собранной информации
   - Включите индекс базы знаний в файл роли

3. **Создание папки знаний**
   - Создайте папку `agents/[agent_name]/knowledge/`
   - Создайте файлы знаний на основе собранной информации
   - Используйте шаблон `management/templates/knowledge.md`

4. **Обновление README.md**
   - **ОБЯЗАТЕЛЬНО**: Добавьте описание нового агента в раздел "Доступные агенты"

**Формат добавления:**
```markdown
### [Название агента]
- **Специализация**: [краткое описание]
- **Когда использовать**: [в каких случаях применять]
- **Основные навыки**: [ключевые возможности]
```

**Пример:**
```markdown
### Marketer
- **Специализация**: Маркетинговые стратегии и кампании
- **Когда использовать**: При планировании маркетинговых активностей, создании контента, анализе рынка
- **Основные навыки**: SMM, контент-маркетинг, аналитика, брендинг
```

## Правила именования

- **Папки агентов**: английские названия в нижнем регистре
- **Файлы знаний**: описательные названия с подчеркиваниями
- **Заголовки в файлах**: четкие и понятные названия разделов

## Структура файлов знаний

Создайте файлы знаний в папке `agents/[agent_name]/knowledge/` на основе собранной информации. Файлы должны содержать знания агента в структурированном виде.

## Важные напоминания

1. **Задавайте подробные вопросы** - не упускайте важные детали
2. **Создавайте файлы знаний** - на основе собранной информации
3. **Всегда обновляйте README.md** при создании нового агента
4. **Следуйте шаблонам** из `management/templates/`
5. **Проверяйте структуру** - убедитесь, что все файлы созданы правильно
6. **Обновляйте даты** в README.md при изменениях

## Шаблоны

Используйте файлы из папки `management/templates/`:
- `role.md` - шаблон для создания роли агента
- `knowledge.md` - шаблон для создания файлов знаний
