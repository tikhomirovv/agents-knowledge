# Refactoring UI - Принципы дизайна интерфейсов

**Источник**: Refactoring UI by Adam Wathan & Steve Schoger

## Основные принципы

### **Начинай с функции, а не с макета**
- **Суть**: При создании нового приложения начинай с конкретной функциональности, а не с общей структуры (навигация, лейаут).
- **Применение**:
  - Определи ключевую функцию (например, "поиск рейсов")
  - Спроектируй только необходимые элементы для этой функции
  - Отложи решения о навигации до проектирования нескольких функций
  - Пример: для поиска рейсов нужны поля "откуда", "куда", "дата вылета", "дата возврата", кнопка поиска
- **Практический совет**: Сначала спроектируй одну функцию полностью, затем переходи к следующей.

### **Детали приходят позже**
- **Суть**: На ранних стадиях не фокусируйся на мелких деталях (шрифты, тени, иконки).
- **Применение**:
  - Используй низкофиделитичные инструменты (бумага + толстый маркер)
  - Игнорируй детали до тех пор, пока не определишь основную структуру
  - Фокусируйся на функциональности, а не на визуальном оформлении
  - Детали важны, но не на этапе исследования идей
- **Практический совет**: Толстый маркер на бумаге не позволит зациклиться на деталях.

### **Работай в градациях серого**
- **Суть**: Сопротивляйся соблазну добавить цвет сразу - сначала работай в градациях серого.
- **Применение**:
  - Используй только оттенки серого для создания иерархии
  - Полагайся на контраст, размер и интервалы для структурирования
  - Цвет добавляй только после того, как иерархия работает в сером
  - Это заставляет создавать более четкую структуру интерфейса
- **Практический совет**: Если интерфейс работает в сером, он будет отлично работать с цветом.

### **Не переусложняй дизайн**
- **Суть**: Не проектируй все функции приложения сразу - работай циклами.
- **Применение**:
  - Проектируй простую версию следующей функции
  - Реализуй её в коде
  - Итерируй на основе реального использования
  - Переходи к следующей функции только после завершения текущей
  - Избегай попыток предугадать все крайние случаи заранее
- **Практический совет**: Строй реальную вещь как можно раньше, чтобы воображение не делало всю работу.

### **Будь пессимистом в планировании**
- **Суть**: Не включай в дизайн функциональность, которую не готов реализовать.
- **Применение**:
  - Проектируй минимальную полезную версию функции
  - Ожидай, что реализация будет сложнее, чем кажется
  - Отложи "приятные дополнения" на потом
  - Сначала строй простую версию, потом улучшай
  - Всегда имей что-то, что можно запустить
- **Практический совет**: Лучше простая система комментариев, чем сложная система, которая никогда не будет готова.

### **Выбери личность дизайна**
- **Суть**: Каждый дизайн имеет личность, которая определяется конкретными факторами.
- **Применение**:
  - **Шрифт**: Serif для элегантности, rounded sans serif для игривости, neutral sans serif для нейтральности
  - **Цвет**: Синий для безопасности, золотой для роскоши, розовый для веселья
  - **Радиус границ**: Маленький радиус нейтрален, большой игрив, отсутствие радиуса серьезно
  - **Язык**: Формальный тон для профессионализма, дружелюбный для близости
  - Изучай сайты целевой аудитории для понимания ожиданий
- **Практический совет**: Оставайся последовательным в выборе элементов личности.

### **Ограничь выбор**
- **Суть**: Миллионы цветов и тысячи шрифтов парализуют принятие решений.
- **Применение**:
  - Создай ограниченную палитру цветов (8-10 оттенков) заранее
  - Определи типографическую шкалу заранее
  - Используй процесс исключения: выбери значение, попробуй соседние, исключи плохие варианты
  - Создавай системы ограничений для всех элементов дизайна
  - Делай сложную работу выбора один раз, а не при каждом решении
- **Практический совет**: Ограниченный выбор делает принятие решений легче и быстрее.

### **Дизайн через процесс исключения**
- **Суть**: При ограниченном наборе значений принятие решений упрощается - меньше "правильных" вариантов.
- **Применение**:
  - Выбери среднее значение (например, 16px для иконки)
  - Попробуй соседние значения (12px и 24px)
  - Исключи явно плохие варианты
  - Если внешний вариант лучше, повтори процесс с ним как средним
  - Работает для любых систем: размеры, цвета, отступы, тени
- **Практический совет**: Когда все варианты заметно отличаются, выбор лучшего становится простым.

### **Систематизируй всё**
- **Суть**: Чем больше систем у тебя есть, тем быстрее работа и меньше сомнений в решениях.
- **Применение**:
  - Создавай системы для: размеров шрифтов, весов, высоты строк, цветов, отступов, размеров, теней, радиусов границ, прозрачности
  - Не определяй всё заранее, но подходи с системным мышлением
  - Ищи возможности вводить новые системы при принятии решений
  - Избегай повторного принятия одних и тех же мелких решений
- **Практический совет**: Системный подход - повторяющаяся тема эффективного дизайна.

### **Иерархия - это всё**
- **Суть**: Визуальная иерархия - самый эффективный инструмент для создания ощущения "дизайна".
- **Применение**:
  - Не все элементы равны - определи важность каждого
  - Намеренно приглушай вторичную и третичную информацию
  - Выделяй самые важные элементы
  - Когда всё конкурирует за внимание, интерфейс кажется шумным и хаотичным
  - Правильная иерархия делает интерфейс приятным даже без изменения цветов и шрифтов
- **Практический совет**: Иерархия важнее поверхностного стиля.

### **Размер - не всё**
- **Суть**: Слишком большая зависимость от размера шрифта для контроля иерархии - ошибка.
- **Применение**:
  - Используй вес шрифта или цвет вместо только размера
  - Делай первичные элементы жирнее, а не больше
  - Используй более мягкий цвет для вторичного текста вместо маленького размера
  - Придерживайся 2-3 цветов: темный для основного, серый для вторичного, светлый серый для третичного
  - Используй 2 веса шрифта: обычный (400-500) и жирный (600-700)
  - Избегай весов меньше 400 для UI
- **Практический совет**: Лучше жирный текст нормального размера, чем большой текст обычного веса.

### **Не используй серый текст на цветных фонах**
- **Суть**: Серый текст работает на белом фоне, но не на цветном - эффект создается снижением контраста.
- **Применение**:
  - На цветном фоне подбирай цвет вручную, а не используй прозрачность
  - Выбирай цвет с тем же оттенком, но с другой насыщенностью и яркостью
  - Избегай белого текста с пониженной прозрачностью - он выглядит тусклым
  - На изображениях прозрачный текст показывает фон через буквы
  - Ручной подбор цвета сохраняет читаемость без потери контраста
- **Практический совет**: Контраст создается близостью к цвету фона, а не серым цветом.

### **Выделяй через приглушение**
- **Суть**: Когда главный элемент не выделяется, приглуши конкурирующие элементы вместо усиления главного.
- **Применение**:
  - Если активный элемент навигации не выделяется, сделай неактивные элементы менее заметными
  - Если боковая панель конкурирует с основным контентом, убери у неё фон
  - Применяй этот принцип к большим частям интерфейса
  - Иногда лучше убрать, чем добавить
- **Практический совет**: Вычитание часто эффективнее сложения для создания фокуса.

### **Метки - последний ресурс**
- **Суть**: При представлении данных избегай формата "метка: значение" - он создает равную важность всех данных.
- **Применение**:
  - **Без меток**: Формат часто говорит сам за себя (email, телефон, цена)
  - **Контекст**: "Customer Support" под именем понятно без метки "Отдел"
  - **Объединение**: "12 left in stock" вместо "In stock: 12"
  - **Вторичные метки**: Если метка нужна, сделай её поддерживающим контентом (меньше, мягче)
  - **Акцент на метках**: На технических страницах пользователи ищут метки, а не значения
- **Практический совет**: Данные важнее меток - стилизуй соответственно.

### **Разделяй визуальную и документную иерархию**
- **Суть**: Семантическая разметка (h1, h2, h3) не должна влиять на визуальную иерархию.
- **Применение**:
  - Используй семантические теги для доступности, но стилизуй для визуальной иерархии
  - Заголовки секций часто действуют как метки, а не как заголовки
  - Содержимое секции важнее её заголовка
  - Заголовки секций должны быть довольно маленькими
  - В крайних случаях скрывай заголовки визуально, оставляя в разметке
- **Практический совет**: Выбирай элементы по семантике, стилизуй по визуальной иерархии.

### **Балансируй вес и контраст**
- **Суть**: Жирный текст выделяется из-за большей площади покрытия - больше пикселей текста на том же пространстве.
- **Применение**:
  - **Контраст для веса**: Иконки "тяжелые" - снижай их контраст для баланса с текстом
  - **Вес для контраста**: Увеличивай толщину тонких границ вместо затемнения цвета
  - Тяжелые элементы приглушай контрастом
  - Легкие элементы подчеркивай весом
  - Баланс создается компенсацией, а не усилением
- **Практический совет**: Понимание веса помогает создавать визуальный баланс.

### **Семантика вторична**
- **Суть**: При проектировании действий на странице иерархия важнее семантики.
- **Применение**:
  - **Первичные действия**: Очевидные, сплошные цвета с высоким контрастом
  - **Вторичные действия**: Понятные, но не заметные - контурные стили или низкий контраст
  - **Третичные действия**: Находимые, но ненавязчивые - стилизуй как ссылки
  - Разрушительные действия не всегда должны быть большими и красными
  - Если разрушительное действие не первичное, дай ему вторичное или третичное оформление
- **Практический совет**: Иерархия действий важнее их семантического значения.

### **Начинай с избытка белого пространства**
- **Суть**: Один из самых простых способов улучшить дизайн - дать элементам больше места для "дыхания".
- **Применение**:
  - Начинай с избытка пространства, затем убирай до нужного результата
  - То, что кажется "слишком много" для отдельного элемента, часто оказывается "в самый раз" в контексте всего интерфейса
  - Элементы получают только минимум пространства, необходимый чтобы не выглядеть плохо
  - Для отличного результата нужно больше белого пространства
  - Плотные интерфейсы имеют место в дашбордах с большим количеством информации
- **Практический совет**: Лучше начать с избытка и убрать, чем добавлять по чуть-чуть.

### **Создай систему отступов и размеров**
- **Суть**: Не выбирай между 120px и 125px - создай ограниченный набор значений заранее.
- **Применение**:
  - Линейная шкала не работает - нужны относительные различия между соседними значениями
  - На малых размерах (иконки, отступы в кнопках) несколько пикселей имеют большое значение (12px→16px = +33%)
  - На больших размерах (ширина карточек) несколько пикселей незаметны (500px→520px = +4%)
  - Значения в шкале не должны быть ближе 25% друг к другу
  - Начни с 16px как базового значения (хорошо делится, размер по умолчанию в браузерах)
  - Малые значения должны быть близко, большие - дальше друг от друга
- **Практический совет**: Система ускоряет работу и создает тонкую консистентность.

### **Не заполняй весь экран**
- **Суть**: Наличие пространства не означает необходимость его использовать.
- **Применение**:
  - Если нужно 600px, используй 600px, а не растягивай на 1200px
  - Растягивание делает интерфейс сложнее для понимания
  - Не делай всё полной ширины только потому, что навигация полной ширины
  - Дай каждому элементу только необходимое пространство
  - Для маленьких интерфейсов на большом холсте - уменьши холст
  - Начни с ~400px для мобильного дизайна
- **Практический совет**: Дополнительное пространство по краям никогда не вредит.

### **Думай колонками**
- **Суть**: Если узкий элемент выглядит несбалансированно в широком интерфейсе, раздели на колонки.
- **Применение**:
  - Вместо расширения формы, вынеси вспомогательный текст в отдельную колонку
  - Это создает баланс без компромиссов в оптимальной ширине
  - Применяй к любым элементам, которые лучше работают в узком формате
  - Не заставляй элементы быть шире, чем нужно
- **Практический совет**: Колонки решают проблему баланса без ухудшения функциональности.

### **Сетки переоценены**
- **Суть**: Сеточные системы полезны, но не все элементы должны быть жидкостными.
- **Применение**:
  - Многие элементы лучше работают с фиксированной шириной, а не относительной
  - Боковая панель должна иметь фиксированную ширину, оптимизированную для содержимого
  - Основной контент может растягиваться, используя внутреннюю сетку
  - Не используй проценты для размеров, если не хочешь масштабирования
  - Не сжимай элемент, пока не нужно - используй max-width
- **Практический совет**: Не будь рабом сетки - дай компонентам нужное пространство.

### **Относительные размеры не масштабируются**
- **Суть**: Не все элементы должны масштабироваться пропорционально друг другу.
- **Применение**:
  - Большие элементы на больших экранах должны сжиматься быстрее маленьких
  - 2.5em может быть идеальным для заголовка на десктопе, но слишком большим на мобильном
  - На мобильном лучше 1.5-1.7x от размера основного текста
  - Разница между маленькими и большими элементами должна быть менее экстремальной на маленьких экранах
  - Откажись от идеи, что всё должно масштабироваться пропорционально
- **Практический совет**: Свобода настройки независимо упрощает дизайн для разных контекстов.

### **Отношения внутри элементов**
- **Суть**: Свойства одного компонента тоже должны масштабироваться независимо.
- **Применение**:
  - Отступы кнопки не должны быть привязаны к размеру шрифта
  - Большие кнопки должны чувствоваться большими, а не просто увеличенными
  - Маленькие кнопки должны чувствоваться маленькими, а не уменьшенными
  - Отступы могут быть более щедрыми на больших размерах и более плотными на маленьких
  - Настройка независимо делает дизайн более естественным
- **Практический совет**: Независимая настройка создает более естественные пропорции.

### **Избегай неоднозначных отступов**
- **Суть**: Когда группы элементов разделены только отступами, должно быть очевидно, какие элементы связаны.
- **Применение**:
  - Между группами должно быть больше пространства, чем внутри группы
  - В формах увеличь отступ между группами полей, чтобы было понятно, какая метка к какому полю относится
  - В статьях добавь больше пространства над заголовками секций
  - В списках отступ между пунктами должен отличаться от высоты строки
  - Применяется как к вертикальным, так и к горизонтальным компоновкам
  - Интерфейсы, которые сложно понять, всегда выглядят хуже
- **Практический совет**: Больше пространства вокруг группы, чем внутри неё.

### **Создай типографическую шкалу**
- **Суть**: Большинство интерфейсов используют слишком много размеров шрифтов - нужна ограниченная система.
- **Применение**:
  - Линейная шкала не работает - нужны разные интервалы для малых и больших размеров
  - Модульные шкалы (4:5, 2:3, золотое сечение) создают дробные значения - избегай их
  - Лучше создавай шкалу вручную для полного контроля
  - Используй px или rem единицы, избегай em (создает неконтролируемые размеры)
  - Пример практичной шкалы: 12px, 14px, 16px, 18px, 20px, 24px, 30px, 36px, 48px, 60px, 72px
- **Практический совет**: Ограниченная шкала ускоряет принятие решений и создает консистентность.

### **Используй хорошие шрифты**
- **Суть**: Выбор качественных шрифтов критичен для профессионального вида интерфейса.
- **Применение**:
  - **Безопасный выбор**: Нейтральные sans-serif как Helvetica или системные шрифты
  - **Системные шрифты**: -apple-system, Segoe UI, Roboto, Noto Sans, Ubuntu, Cantarell, Helvetica Neue
  - **Фильтр по весам**: Игнорируй шрифты с менее чем 5 весами (показывает качество)
  - **Оптимизация читаемости**: Избегай сжатых шрифтов с короткой x-height для основного текста
  - **Популярность**: Сортируй по популярности - популярные шрифты обычно качественные
  - **Изучай лучших**: Инспектируй шрифты на хорошо спроектированных сайтах
- **Практический совет**: Доверяй коллективному выбору тысяч дизайнеров.

### **Контролируй длину строк**
- **Суть**: Оптимальная длина строк критична для читаемости - не подгоняй текст под макет.
- **Применение**:
  - Оптимальная длина: 45-75 символов на строку
  - Используй ширину 20-35em для достижения нужной длины
  - Шире 75 символов - рискованная территория
  - Даже при широком контенте ограничивай ширину параграфов
  - Разная ширина в одной области контента выглядит более отполированно
- **Практический совет**: Создавай лучший опыт чтения, а не подгоняй под макет.

### **Выравнивай по базовой линии, а не по центру**
- **Суть**: При смешивании размеров шрифтов выравнивай по базовой линии, а не по центру.
- **Применение**:
  - Базовая линия - воображаемая линия, на которой "стоят" буквы
  - Выравнивание по базовой линии использует естественную точку отсчета
  - Центрирование создает неловкое смещение базовых линий
  - Особенно заметно при близком расположении текста разного размера
  - Результат: более простой и чистый вид
- **Практический совет**: Базовая линия - естественная точка отсчета для глаз.

### **Высота строк пропорциональна**
- **Суть**: Высота строк должна учитывать длину строк и размер шрифта.
- **Применение**:
  - **Длина строк**: Узкий контент - короткая высота строк (1.5), широкий - длинная (до 2)
  - **Размер шрифта**: Маленький текст - больше высота строк, большой - меньше
  - Обратная пропорциональность: высота строк и размер шрифта
  - Большие заголовки могут иметь высоту строк 1 (без дополнительного пространства)
  - Цель: легко найти следующую строку при переносе
- **Практический совет**: Высота строк и размер шрифта обратно пропорциональны.

### **Не каждой ссылке нужен цвет**
- **Суть**: В интерфейсах, где почти всё - ссылки, не все должны быть ярко выделены.
- **Применение**:
  - В параграфах ссылки должны выделяться и выглядеть кликабельными
  - В интерфейсах с множеством ссылок используй более тонкие методы
  - Подчеркивание или изменение цвета только при наведении для второстепенных ссылок
  - Используй более жирный вес или темный цвет для важных ссылок
  - Не конкурируй за внимание с более важными действиями
- **Практический совет**: Ссылки должны быть находимыми, но не конкурировать за внимание.

### **Выравнивай с учетом читаемости**
- **Суть**: Выравнивание текста должно соответствовать направлению языка и контексту.
- **Применение**:
  - **Основной текст**: Левый край для английского и большинства языков
  - **Центрирование**: Только для заголовков или коротких блоков (2-3 строки)
  - **Длинный текст**: Всегда левое выравнивание, лучше сократить контент
  - **Числа в таблицах**: Правое выравнивание для легкого сравнения
  - **Выравнивание по ширине**: Используй переносы слов, иначе будут некрасивые пробелы
- **Практический совет**: Читаемость важнее визуальной симметрии.

### **Используй межбуквенные интервалы эффективно**
- **Суть**: Межбуквенные интервалы можно настраивать для улучшения дизайна в определенных ситуациях.
- **Применение**:
  - **Общее правило**: Доверяй дизайнеру шрифта, не меняй интервалы
  - **Заголовки**: Уменьшай интервалы для имитации сжатого вида заголовочных шрифтов
  - **Заглавные буквы**: Увеличивай интервалы для улучшения читаемости
  - **Заголовочные шрифты**: Не используй для малых размеров даже с увеличенными интервалами
  - **Заглавные буквы**: Менее разнообразны визуально, нужны большие интервалы
- **Практический совет**: Межбуквенные интервалы - тонкая настройка для особых случаев.

### **Откажись от hex в пользу HSL**
- **Суть**: HSL представляет цвета через атрибуты, которые интуитивно воспринимает человеческий глаз.
- **Применение**:
  - **Hue (оттенок)**: Позиция на цветовом круге (0°=красный, 120°=зеленый, 240°=синий)
  - **Saturation (насыщенность)**: Насколько яркий цвет (0%=серый, 100%=интенсивный)
  - **Lightness (светлота)**: Близость к черному или белому (0%=черный, 50%=чистый цвет, 100%=белый)
  - HSL vs HSB: В HSB яркость работает по-другому, браузеры понимают только HSL
  - Цвета с общими визуальными характеристиками выглядят похоже в коде
- **Практический совет**: HSL делает работу с цветом более интуитивной и предсказуемой.

### **Тебе нужно больше цветов, чем ты думаешь**
- **Суть**: Пять "идеальных" цветов недостаточно для создания реального интерфейса.
- **Применение**:
  - **Серые**: 8-10 оттенков для текста, фонов, панелей, форм (начни с темно-серого, не черного)
  - **Основные цвета**: 1-2 цвета для главных действий, 5-10 оттенков каждого
  - **Акцентные цвета**: Для выделения функций, семантических состояний (красный, желтый, зеленый)
  - **Сложные интерфейсы**: До 10 разных цветов с 5-10 оттенками каждый
  - Избегай генераторов палитр - они создают слишком ограниченные наборы
- **Практический совет**: Планируй комплексную палитру заранее, а не выбирай по 5 цветов.

### **Определи оттенки заранее**
- **Суть**: Не создавай оттенки на лету с помощью CSS-функций - определи фиксированный набор.
- **Применение**:
  - **Базовый цвет**: Выбери средний оттенок, который хорошо работает как фон кнопки
  - **Края**: Определи самый темный (для текста) и самый светлый (для фонов) оттенки
  - **Заполнение пробелов**: Используй подход "900-500-100", заполняй промежуточные значения
  - **9 оттенков**: Идеальное количество - легко делить и заполнять пробелы
  - **Серые**: Начни с краев (темный текст, светлый фон), заполни промежутки
  - Доверяй глазам, а не математике - корректируй по необходимости
- **Практический совет**: Фиксированный набор оттенков предотвращает хаос из 35 похожих синих.

### **Не позволяй светлоте убить насыщенность**
- **Суть**: В HSL при приближении к 0% или 100% светлоты влияние насыщенности ослабевает.
- **Применение**:
  - Увеличивай насыщенность по мере удаления от 50% светлоты
  - Используй воспринимаемую яркость: разные оттенки имеют разную естественную яркость
  - **Яркие оттенки**: Желтый, голубой, пурпурный (60°, 180°, 300°)
  - **Темные оттенки**: Красный, зеленый, синий (0°, 120°, 240°)
  - **Изменение яркости через оттенок**: Поворачивай к ближайшему яркому/темному оттенку
  - Не поворачивай больше чем на 20-30° - иначе получишь другой цвет
- **Практический совет**: Комбинируй изменение оттенка и светлоты для богатых, не тусклых цветов.

### **Серые не обязаны быть серыми**
- **Суть**: Насыщенные серые создают температурные ощущения и выглядят более естественно.
- **Применение**:
  - **Холодные серые**: Насыщай синим для прохладного ощущения
  - **Теплые серые**: Насыщай желтым или оранжевым для теплого ощущения
  - **Консистентность**: Увеличивай насыщенность для светлых и темных оттенков
  - **Степень насыщенности**: От легкого намека до сильного наклона в сторону
  - Истинный серый (0% насыщенности) выглядит неестественно
- **Практический совет**: Насыщенные серые создают более живой и приятный интерфейс.

### **Доступность не должна быть уродливой**
- **Суть**: WCAG требования (4.5:1 для обычного текста, 3:1 для крупного) можно выполнить элегантно.
- **Применение**:
  - **Переворот контраста**: Темный текст на светлом фоне вместо светлого на темном
  - **Поворот оттенка**: Для цветного текста на цветном фоне поворачивай к более яркому оттенку
  - **Иерархия**: Темные цветные фоны привлекают внимание - используй осторожно
  - **Цветной текст**: Поворачивай к голубому, пурпурному или желтому для лучшего контраста
  - Доступность не означает отказ от цвета
- **Практический совет**: Инвертируй контраст и поворачивай оттенки для элегантной доступности.

### **Не полагайся только на цвет**
- **Суть**: Цвет должен поддерживать информацию, а не быть единственным способом её передачи.
- **Применение**:
  - **Дублирование информации**: Добавляй иконки к цветовым индикаторам
  - **Контраст вместо цвета**: Для графиков используй светлые и темные варианты
  - **Цветовая слепота**: 8% мужчин имеют красно-зеленую слепоту
  - **Поддержка, не замена**: Цвет должен усиливать то, что дизайн уже говорит
  - **Тестирование**: Проверяй интерфейс в оттенках серого
- **Практический совет**: Цвет - это усилитель информации, а не её носитель.

### **Имитируй источник света**
- **Суть**: Свет падает сверху - это фундаментальное правило для создания ощущения глубины.
- **Применение**:
  - **Поднятые элементы**: Верхний край светлее (получает больше света), нижний темнее
  - **Углубленные элементы**: Верхний край темнее (тень от выступа), нижний светлее
  - **Естественный взгляд**: Люди смотрят слегка вниз на экраны - показывай верхний край, скрывай нижний
  - **Ручной подбор цвета**: Не используй полупрозрачный белый - он высасывает насыщенность
  - **Блокировка света**: Поднятые элементы создают тень снизу
- **Практический совет**: Имитируй, как свет взаимодействует с формами в реальном мире.

### **Используй тени для передачи высоты**
- **Суть**: Тени позиционируют элементы на виртуальной z-оси для создания осмысленного ощущения глубины.
- **Применение**:
  - **Малые тени**: Элемент слегка приподнят (кнопки)
  - **Средние тени**: Элемент выше остального UI (выпадающие меню)
  - **Большие тени**: Элемент привлекает внимание (модальные окна)
  - **Система высот**: Определи 5 вариантов теней от малых до больших
  - **Интерактивность**: Тени реагируют на взаимодействие (нажатие, перетаскивание)
  - **Фокус**: Чем ближе к пользователю, тем больше привлекает внимание
- **Практический совет**: Думай о позиции на z-оси, а не о самой тени.

### **Тени могут состоять из двух частей**
- **Суть**: Комбинирование двух теней дает больше контроля и реалистичности.
- **Применение**:
  - **Первая тень**: Большая, мягкая, с большим смещением - имитирует тень от прямого света
  - **Вторая тень**: Маленькая, темная, с малым смещением - имитирует область под объектом
  - **Высота**: При увеличении высоты вторая тень становится менее заметной
  - **Контроль**: Можешь держать большую тень тонкой, а малую - четкой
  - **Реализм**: Имитирует реальное поведение света и теней
- **Практический совет**: Две тени создают более реалистичный и контролируемый эффект.

### **Даже плоские дизайны могут иметь глубину**
- **Суть**: Плоский дизайн не означает отсутствие глубины - просто другие способы её создания.
- **Применение**:
  - **Цвет для глубины**: Светлые объекты кажутся ближе, темные - дальше
  - **Поднятые элементы**: Светлее фона
  - **Углубленные элементы**: Темнее фона
  - **Сплошные тени**: Короткие тени без размытия для плоской эстетики
  - **Наложение элементов**: Создание слоев через пересечение
  - **Невидимые границы**: Цветные границы, совпадающие с фоном
- **Практический совет**: Глубина возможна без градиентов и размытых теней.

### **Наложение элементов создает слои**
- **Суть**: Один из самых эффективных способов создания глубины - наложение элементов.
- **Применение**:
  - **Карточки**: Смещай карточку так, чтобы она пересекала границу между фонами
  - **Высота**: Делай элемент выше родителя для наложения с обеих сторон
  - **Компоненты**: Применяй к маленьким элементам (элементы управления карусели)
  - **Изображения**: Используй "невидимые границы" для избежания конфликтов
  - **Слои**: Создавай ощущение множественных слоев без теней
- **Практический совет**: Наложение создает глубину без сложных эффектов.

### **Не увлекайся реализмом**
- **Суть**: Понимание света полезно, но не нужно делать интерфейсы фотореалистичными.
- **Применение**:
  - **Умеренность**: Заимствуй визуальные подсказки из реального мира, но не всё
  - **Четкость**: Слишком реалистичные эффекты могут сделать интерфейс загруженным
  - **Функциональность**: Глубина должна помогать пониманию, а не отвлекать
  - **Простота**: Лучше простые, понятные эффекты, чем сложные
  - **Фокус**: Эффекты должны поддерживать иерархию, а не конкурировать с ней
- **Практический совет**: Заимствуй из реального мира, но не копируй полностью.

### **Используй хорошие фотографии**
- **Суть**: Плохие фотографии испортят дизайн, даже если всё остальное выглядит отлично.
- **Применение**:
  - **Профессиональный фотограф**: Для специфических потребностей проекта
  - **Качественный сток**: Для общих потребностей (Unsplash, платные ресурсы)
  - **Не используй заглушки**: Не планируй заменить их снимками со смартфона
  - **Качество важнее**: Хорошие фото требуют навыков освещения, композиции, цвета
  - **Инвестиции**: Плохие фото - это потраченные впустую усилия на дизайн
- **Практический совет**: Хорошие фото - это инвестиция в качество всего дизайна.

### **Текст нуждается в консистентном контрасте**
- **Суть**: Проблема с читаемостью текста на фоновых изображениях - в динамике самого изображения.
- **Применение**:
  - **Наложение**: Полупрозрачное наложение для снижения динамики изображения
  - **Снижение контраста**: Уменьшение контраста самого изображения
  - **Окрашивание**: Одноцветное окрашивание изображения (контраст → обесцвечивание → заливка)
  - **Тень текста**: Большое размытие без смещения для тонкого свечения
  - **Комбинирование**: Сочетание методов для лучшего результата
- **Практический совет**: Снижай динамику изображения, а не меняй цвет текста.

### **У всего есть предназначенный размер**
- **Суть**: Масштабирование элементов за пределы их предназначенного размера ухудшает качество.
- **Применение**:
  - **Иконки**: Не увеличивай иконки, предназначенные для 16-24px
  - **Контейнеры**: Помещай маленькие иконки в цветные контейнеры
  - **Скриншоты**: Делай скриншоты в меньшем размере, не сжимай большие
  - **Частичные скриншоты**: Показывай только нужную часть
  - **Упрощенные версии**: Рисуй упрощенные версии для маленьких размеров
  - **Favicon**: Создавай специальную версию для 16px, не сжимай логотип
- **Практический совет**: Создавай элементы в нужном размере, не масштабируй.

### **Остерегайся пользовательского контента**
- **Суть**: Пользовательские изображения могут нарушить дизайн - контролируй их отображение.
- **Применение**:
  - **Фиксированные контейнеры**: Центрируй изображения в контейнерах фиксированного размера
  - **Обрезка**: Используй background-size: cover для обрезки лишнего
  - **Предотвращение слияния**: Используй тонкую внутреннюю тень вместо границ
  - **Полупрозрачные границы**: Альтернатива теням для разделения
  - **Контроль формы**: Не позволяй пользователям нарушать структуру страницы
- **Практический совет**: Контролируй отображение пользовательского контента.

### **Усиливай стандартные элементы**
- **Суть**: Не всегда нужно добавлять новые элементы - улучшай то, что уже есть.
- **Применение**:
  - **Списки**: Заменяй маркеры иконками (галочки, стрелки, специфичные иконки)
  - **Цитаты**: Увеличивай размер и меняй цвет кавычек
  - **Ссылки**: Меняй цвет, вес шрифта, добавляй кастомные подчеркивания
  - **Формы**: Кастомные чекбоксы и радиокнопки с цветами бренда
  - **Простота**: Часто достаточно одного цвета бренда для полированного вида
- **Практический совет**: Улучшай существующие элементы вместо добавления новых.

### **Добавляй цвет акцентными границами**
- **Суть**: Цветные акцентные границы - простой способ добавить визуальную изюминку.
- **Применение**:
  - **Карточки**: Цветная полоса сверху
  - **Навигация**: Подсветка активных элементов
  - **Алерты**: Цветная полоса сбоку
  - **Заголовки**: Короткий акцент снизу
  - **Макет**: Цветная полоса сверху всего макета
  - **Простота**: Не требует графических навыков - просто цветной прямоугольник
- **Практический совет**: Цветные границы делают дизайн более "дизайнерским".

### **Украшай фоны**
- **Суть**: Даже при хорошей иерархии и типографике дизайн может казаться скучным.
- **Применение**:
  - **Изменение цвета**: Для выделения панелей или разделов страницы
  - **Градиенты**: Используй оттенки не более чем на 30° друг от друга
  - **Повторяющиеся паттерны**: Тонкие паттерны с низким контрастом
  - **Простые формы**: Геометрические фигуры в определенных позициях
  - **Иллюстрации**: Упрощенные графические элементы
  - **Низкий контраст**: Чтобы не мешать читаемости контента
- **Практический совет**: Фоновые украшения должны поддерживать, а не отвлекать.

### **Не игнорируй пустые состояния**
- **Суть**: Пустые состояния - первое взаимодействие пользователя с функцией, делай их интересными.
- **Применение**:
  - **Приоритет**: Планируй пустые состояния, не оставляй на потом
  - **Визуалы**: Добавляй изображения или иллюстрации для привлечения внимания
  - **Призыв к действию**: Подчеркивай следующий шаг для пользователя
  - **Скрытие UI**: Убирай неактивные табы и фильтры
  - **Возможность**: Используй как шанс быть интересным и захватывающим
- **Практический совет**: Пустые состояния - это возможность произвести первое впечатление.

### **Используй меньше границ**
- **Суть**: Границы - не единственный способ разделения элементов, их избыток создает загруженность.
- **Применение**:
  - **Тени**: Box shadows более тонкие, чем границы
  - **Разные фоны**: Слегка разные цвета фона создают разделение
  - **Дополнительные отступы**: Увеличение пространства между элементами
  - **Комбинирование**: Убирай границы, если уже используешь разные фоны
  - **Альтернативы**: Ищи способы разделения без границ
- **Практический совет**: Границы должны быть последним выбором для разделения.

### **Думай нестандартно**
- **Суть**: Не ограничивайся предвзятыми представлениями о том, как должны выглядеть компоненты.
- **Применение**:
  - **Выпадающие меню**: Разделяй на секции, используй колонки, добавляй иконки
  - **Таблицы**: Объединяй связанные колонки, добавляй изображения и цвета
  - **Радиокнопки**: Заменяй на выбираемые карточки
  - **Свобода**: Компоненты - это просто элементы на экране, делай с ними что хочешь
  - **Инновации**: Иногда свобода нужна для выхода на новый уровень
- **Практический совет**: Выходи за рамки привычных представлений о компонентах.

### **Ищи решения, которые не пришли бы в голову**
- **Суть**: Изучай дизайны, которые нравятся, и анализируй неочевидные решения.
- **Применение**:
  - **Вопрос**: "Сделал ли дизайнер что-то, что я никогда бы не подумал сделать?"
  - **Примеры**: Инвертированные цвета, нестандартное позиционирование, разные цвета в заголовке
  - **Анализ**: Обращай внимание на неинтуитивные решения
  - **Применение**: Используй найденные идеи в своих проектах
  - **Постоянное обучение**: Дизайн - это постоянное изучение новых подходов
- **Практический совет**: Анализируй работы, которые вдохновляют, и находи неочевидные решения.

### **Воссоздавай любимые интерфейсы**
- **Суть**: Лучший способ заметить детали, делающие дизайн полированным - воссоздать его с нуля.
- **Применение**:
  - **Без подглядывания**: Не используй инструменты разработчика
  - **Сравнение**: Сравнивай свою версию с оригиналом
  - **Открытия**: Самостоятельно находишь трюки (высота строк, межбуквенные интервалы, тени)
  - **Детали**: Замечаешь мелкие детали, которые делают дизайн профессиональным
  - **Навыки**: Постоянно изучай работы, которые вдохновляют
- **Практический совет**: Воссоздание дизайнов - лучший способ изучить профессиональные техники.
